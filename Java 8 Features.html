1. Lambda Expressions

↑ Back to Features

Introduction

A Lambda Expression is a short, anonymous function (without a name) that can be used to implement methods of functional interfaces (interfaces with only one abstract method), enabling concise code and reducing boilerplate.

  • First used in LISP programming language
  • Main objective: Bring benefits of functional programming into Java
  • Anonymous function - Function without name, modifiers and return type
  • Braces required for multiple lines
  • Parentheses optional for single parameter

Syntax

Java
(parameters) -> { body }

Benefits

Less Code

Reduces need for boilerplate like anonymous classes

Readability

More concise and expressive code

Functional Style

Encourages declarative programming

Parallelism Friendly

Works well with Streams and parallel operations

Example: Calculator using Lambda

This example demonstrates how to implement a simple calculator using lambda expressions with a functional interface.

Java
public class LambdaCalculator {
    public static void main(String[] args) {
        Calculator add = (a, b) -> a + b;
        Calculator subtract = (a, b) -> a - b;

        System.out.println("Addition: " + add.operation(10, 5));
        System.out.println("Subtraction: " + subtract.operation(10, 5));
    }
}

@FunctionalInterface
interface Calculator {
    int operation(int a, int b);
}

Output:

Output
Addition: 15
Subtraction: 5

Key Components:

Component Description
@FunctionalInterface Annotation marking an interface with exactly one abstract method
(a, b) -> a + b Lambda expression implementing the addition operation
(a, b) -> a - b Lambda expression implementing the subtraction operation
operation() The single abstract method in the functional interface

Functional Interface

The Calculator interface has exactly one abstract method, making it a functional interface that can be implemented with lambdas.

Lambda Benefits

The lambda expressions provide concise implementations of the operation method without needing concrete classes.

How It Works:

  1. The Calculator interface defines a single operation method
  2. Lambda expressions provide implementations for addition and subtraction
  3. Each lambda is assigned to a Calculator variable
  4. The operations are invoked through the interface method
  5. The compiler infers parameter types from the interface

Example: Sorting using Lambda

This example demonstrates how to sort a list of strings using lambda expressions with a custom comparator.

Java
import java.util.*;

public class LambdaSort {
    public static void main(String[] args) {
        List<String> names = Arrays.asList(
            "John", "Akhil", "Zara", "Ben"
        );

        // Sort using lambda
        names.sort((a, b) -> a.compareToIgnoreCase(b));

        System.out.println("Sorted Names: " + names);
    }
}

Output:

Output
Sorted Names: [Akhil, Ben, John, Zara]

Explanation:

  • The List.sort() method takes a Comparator as parameter
  • Instead of creating an anonymous class, we use a lambda expression
  • (a, b) -> a.compareToIgnoreCase(b) implements the comparison logic
  • The lambda replaces the need for explicit compare() method implementation
  • Case-insensitive sorting is achieved with compareToIgnoreCase()

2. Functional Interfaces

↑ Back to Features

What is a Functional Interface?

A Functional Interface (FI) is an interface that contains exactly one abstract method. It can have multiple default or static methods, but only one abstract method.

Common Functional Interfaces

Interface Method
Runnable void run()
Comparable<T> int compareTo(T o)
Comparator<T> int compare(T o1, T o2)
Callable<V> V call()
ActionListener void actionPerformed(ActionEvent e)

@FunctionalInterface Annotation

  • Explicitly declares that an interface is functional
  • Prevents accidental addition of abstract methods
  • Not mandatory but recommended

1. Lambda Expression with Functional Interface

Lambda expressions provide a concise syntax to implement the single abstract method of functional interfaces.

Java
@FunctionalInterface
interface Transaction {
    void perform(double amount);
}

public class BankSystem {
    public static void main(String[] args) {

        // Lambda implementation
        Transaction deposit = (amount) -> {
            System.out.println("Deposited: $" + amount);
        };

        Transaction withdraw = (amount) -> {
            System.out.println("Withdrawn: $" + amount);
        };

        processTransaction(deposit, 1000);
        processTransaction(withdraw, 500);
    }

    public static void processTransaction(Transaction t, double amount) {
        System.out.println("Transaction initiated...");
        t.perform(amount);
        System.out.println("Transaction completed.\n");
    }
}

3. Default and Static Methods

↑ Back to Features

Can we have default/static methods in functional interfaces?

Yes! A functional interface can have exactly one abstract method and any number of default and static methods.

Use Case: Payment Gateway System

A flexible payment system supporting multiple payment methods with shared functionality using functional interfaces.

Java
@FunctionalInterface
public interface PaymentGateway {

    // Single Abstract Method
    void pay(double amount);

    // Default method (can be overridden)
    default void logTransaction(String method, double amount) {
        System.out.println("Log: Payment of ₹" + amount + " via " + method);
    }

    // Static method (utility)
    static boolean validateAmount(double amount) {
        return amount > 0;
    }
}
Java
public class PaymentApp {
    public static void main(String[] args) {
        double amount = 1500.0;

        if (PaymentGateway.validateAmount(amount)) {

            // Credit Card Payment using Lambda
            PaymentGateway creditCardPayment = (amt) -> {
                System.out.println("Paid " + amt + " using Credit Card");
            };

            // Wallet Payment using Anonymous Inner Class
            PaymentGateway walletPayment = new PaymentGateway() {
                @Override
                public void pay(double amt) {
                    System.out.println(" Paid ₹" + amt + " using Wallet");
                }

                @Override
                public void logTransaction(String method, double amt) {
                    System.out.println(" Custom Log: Wallet Payment of ₹" + amt);
                }
            };

            // Execute payments
            creditCardPayment.logTransaction("Credit Card", amount);
            creditCardPayment.pay(amount);

            walletPayment.logTransaction("Wallet", amount);
            walletPayment.pay(amount);

        } else {
            System.out.println("Invalid amount");
        }
    }
}

Paid ₹1500.0 using Wallet

4. Predefined Functional Interfaces

↑ Back to Features

These interfaces from java.util.function package provide common functional patterns for lambda expressions.

1. Predicate<T>

Represents a boolean-valued function that takes one argument and returns true/false.

Java
import java.util.function.Predicate;

public class PredicateExample {
    public static void main(String[] args) {
        Predicate<Integer> isAdult = age -> age >= 18;

        System.out.println("Is 20 an adult? " + isAdult.test(20)); // true
        System.out.println("Is 15 an adult? " + isAdult.test(15)); // false
    }
}

Key Features:

  • Single abstract method: boolean test(T t)
  • Commonly used for filtering and validation
  • Default methods: and(), or(), negate()

2. Function<T, R>

Represents a function that accepts one argument and produces a result.

Java
import java.util.function.Function;

public class FunctionExample {
    public static void main(String[] args) {
        Function<Double, Double> celsiusToFahrenheit = c -> (c * 9/5) + 32;

        System.out.println("25°C in Fahrenheit: " + celsiusToFahrenheit.apply(25.0)); // 77.0
    }
}

Key Features:

  • Single abstract method: R apply(T t)
  • Used for transformations and conversions
  • Default methods: compose(), andThen()

3. Consumer<T>

Represents an operation that accepts a single input and returns no result.

Java
import java.util.function.Consumer;

public class ConsumerExample {
    public static void main(String[] args) {
        Consumer<String> printUpperCase = name -> System.out.println(name.toUpperCase());

        printUpperCase.accept("Akhil"); // Output: AKHIL
    }
}

Key Features:

  • Single abstract method: void accept(T t)
  • Used for side-effects like printing, saving, etc.
  • Default method: andThen()

4. Supplier<T>

Represents a supplier of results that takes no arguments.

Java

  • Single abstract method: T get()

5. BiFunction<T, U, R>

Represents a function that accepts two arguments and produces a result.

Java
import java.util.function.BiFunction;

public class BiFunctionExample {
    public static void main(String[] args) {

Key Features:

  • Single abstract method: R apply(T t, U u)
  • Two-argument version of Function
  • Used for operations requiring two inputs

6. UnaryOperator<T>

Represents an operation on a single operand that produces a result of the same type.

Java
import java.util.function.UnaryOperator;

public class UnaryOperatorExample {
    public static void main(String[] args) {

}

Key Features:

  • Special case of Function where input and output types are same
  • Single abstract method: T apply(T t)
  • Used for modifications and transformations of the same type

7. BinaryOperator<T>

Represents an operation upon two operands of the same type, producing a result of the same type.

Java
import java.util.function.BinaryOperator;

public class BinaryOperatorExample {

Key Features:

  • Special case of BiFunction where all types are same

  • Used for mathematical operations and reductions

Example: Shopping Checkout

This implementation demonstrates an online shopping checkout system using Java's functional programming features.

Functional Interfaces Used

Interface Purpose Implementation
Predicate<Double> Cart validation Check if total > 0
Function<Double, Double> Apply discount 10% discount calculation
UnaryOperator<Double> Add tax 18% tax calculation
Supplier<String> Generate invoice Create unique invoice ID
Consumer<String> Print bill Output formatted receipt

Implementation Code

Java
import java.util.function.*;
import java.util.UUID;

public class ShoppingCheckout {

    public static void main(String[] args) {
        double cartTotal = 1200.0;

        // 1. Predicate: Check if cart is valid (non-zero total)
        Predicate<Double> isCartValid = total -> total > 0;

        // 2. Function: Apply 10% discount
        Function<Double, Double> applyDiscount = total -> total - (total * 0.10);

        // 3. UnaryOperator: Add 18% tax
        UnaryOperator<Double> addTax = discountedTotal ->
            discountedTotal + (discountedTotal * 0.18);

        // 4. Supplier: Generate unique invoice ID
        Supplier<String> generateInvoiceId = () ->
            "INV-" + UUID.randomUUID().toString().substring(0, 8).toUpperCase();

        // 5. Consumer: Print final bill
        Consumer<String> printBill = bill -> {
            System.out.println("----- BILL SUMMARY -----");
            System.out.println(bill);
            System.out.println("------------------------");
        };

        // Process Checkout
        if (isCartValid.test(cartTotal)) {
            double discountedTotal = applyDiscount.apply(cartTotal);
            double finalAmount = addTax.apply(discountedTotal);
            String invoice = generateInvoiceId.get();

            StringBuilder bill = new StringBuilder();
            bill.append("Invoice ID : ").append(invoice).append("\n");
            bill.append("Cart Total : ₹").append(cartTotal).append("\n");
            bill.append("Discounted : ₹").append(discountedTotal).append("\n");
            bill.append("Final Amount (with tax): ₹").append(finalAmount);

            printBill.accept(bill.toString());
        } else {
            System.out.println("Cart is empty. Cannot proceed to checkout.");
        }
    }
}

Invoice ID : INV-7A3F9B2C
Cart Total : ₹1200.0
Discounted : ₹1080.0
Final Amount (with tax): ₹1274.4

Key Benefits

  • Each operation is encapsulated in its own functional component
  • Clear separation of concerns makes the code more maintainable
  • Easy to modify or extend individual steps without affecting others
  • Functional programming approach results in more declarative code

5. Double Colon Operator (::)

↑ Back to Features

The double colon operator :: is called the method reference operator in Java. It's used to refer to methods or constructors without invoking them — making code more concise and readable.

Types of Method References

Type Syntax Use Case

Instance Method of Object objectRef::instanceMethod When referencing a method of an object

Real-Time Example: Order Processing System

This example demonstrates method references and constructor references in Java for processing orders.

Key Operations

Step 1: Define Order and Customer Classes

public class Order {

    private String status;

        this.status = status;

    }

    public String getStatus() {
        return status;
    }

    public static boolean isDelivered(Order o) {

    public Customer(String name) {
        this.name = name;
        System.out.println("Customer created: " + name);
    }

Step 2: Using Method and Constructor References

Java
import java.util.*;
import java.util.stream.*;

public class OrderApp {

            new Order(101, "Processing"),

            new Order(103, "Delivered")
        );

        // 1. Instance method reference (Order::print)

            .map(Customer::new) // constructor reference
            .collect(Collectors.toList());
    }

6. Streams

↑ Back to Features

Streams in Java provide a declarative way to process data collections like Lists, Sets, and Maps. They help perform complex data manipulations using functional-style operations.

Key Features

  • Functional - Supports lambda expressions
  • Lazy - Operations are evaluated only when necessary
  • Pipeline - Operations can be chained
  • Parallelizable - Can easily use .parallelStream() for performance
  • Immutable - Does not change the source; returns a new result

Stream Pipeline Structure

Collection.stream() → Intermediate Operations → Terminal Operation

  • Intermediate Operations: Return a stream (e.g., map(), filter(), sorted())
  • Terminal Operations: Return a result (e.g., collect(), forEach(), count())

Real-Time Stream API Use Cases

Practical examples demonstrating Java Stream API operations with real-world scenarios.

  • Filtering data

  • Aggregating values

  • Grouping data

1. Filter Employees Earning > 50,000

Java
import java.util.*;
import java.util.stream.*;

class Employee {
    String name;
    double salary;

    public Employee(String name, double salary) {
        this.name = name;
        this.salary = salary;
    }

    public String toString() {
        return name + ": " + salary;
    }
}

public class FilterExample {
    public static void main(String[] args) {
        List<Employee> employees = List.of(
            new Employee("Rohit", 70000),
            new Employee("Vishnu", 45000),
            new Employee("Akhil", 60000)
        );

        List<Employee> highEarners = employees.stream()

Old API Issue Java 8 Solution

8. Optional Class

↑ Back to Features

The Optional class in Java is a container object used to represent the presence or absence of a value. It helps prevent NullPointerException.

Real-Time Scenario: User Profile Lookup with Optional

Example demonstrating how Java's Optional can elegantly handle user profile lookups in an application.

Key Benefits

  • Eliminates null checks
  • Prevents NullPointerException
  • Expressive API for handling presence/absence
  • Cleaner, more readable code

Step 1: Define User Class

Java
public class User {
    private String email;
    private String name;

    public User(String email, String name) {
        this.email = email;
        this.name = name;
    }

    public String getEmail() { return email; }
    public String getName() { return name; }

    @Override
    public String toString() {
        return name + " (" + email + ")";
    }
}

Step 2: UserService with Optional Logic

Java
import java.util.*;

public class UserService {
    private static final List<User> users = List.of(
        new User("akhil@example.com", "Akhil Kotti"),
        new User("vishnu@example.com", "Vishnu Sammeta")
    );

    public Optional<User> findUserByEmail(String email) {
        return users.stream()
            .filter(u -> u.getEmail().equalsIgnoreCase(email))
            .findFirst(); //Returns Optional<User>
    }
}

Step 3: Client Code Using Optional

Java
public class OptionalExampleApp {
    public static void main(String[] args) {
        UserService service = new UserService();

        // Try with existing email
        String inputEmail1 = "akhil@example.com";
        Optional<User> result1 = service.findUserByEmail(inputEmail1);
        result1.ifPresent(user -> System.out.println("Found: " + user));
        // orElse() fallback
        System.out.println("User: " + result1.orElse(new User("unknown", "Guest")));

        // Try with non-existing email
        String inputEmail2 = "noone@example.com";
        Optional<User> result2 = service.findUserByEmail(inputEmail2);
        result2.ifPresentOrElse(
            user -> System.out.println("Found: " + user),
            () -> System.out.println("User not found for " + inputEmail2)
        );
    }
}

Output

			User: Akhil Kotti (akhil@example.com)
			User not found for noone@example.com

Optional vs Traditional Null Handling

Without Optional With Optional
Tedious null checks Clean and expressive handling
Prone to NullPointerException Prevents NPE by design
if (user != null) everywhere .ifPresent(), .orElse(), .map() etc.

Common Optional Methods

Method Description
Optional.of() Wraps a non-null value
Optional.ofNullable() Wraps a value that might be null
isPresent() Checks if value is present
ifPresent(Consumer) Executes block if value exists
orElse(default) Returns default if value is absent
orElseGet(Supplier) Lazy default value
orElseThrow() Throws exception if no value
map(Function) Transforms the value

Best Practices

  • Use Optional as return type for methods that might not return a value
  • Avoid using Optional for fields, method parameters, or collections
  • Prefer ifPresent() over isPresent() + get()
  • Use orElseGet() instead of orElse() for expensive defaults
  • Consider orElseThrow() when absence should be exceptional
